    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-lazymap---auto-nmap-scanning-script" class="anchor" aria-hidden="true" href="#lazymap---auto-nmap-scanning-script"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>LazyMap - Auto NMAP Scanning Script</h1>
<p>WARNING: Significant divergence from original LazyMap, please read the lazymap.sh preamble and script very carefully:</p>
<p>This fork is a heavily modified version for running enumeration:
  nmap
  nmap NSEs
  nikto
  enum4linux
  onesixtyone
  snmp walk

scanner_nse.sh spawns off a bunch of NSE scanner scripts that may step all over each other, YMMV.
shell_spawner.sh also uses msfvenom to spit out a number of common shells to your interface's IP.

This is a quick/dity modification of the original lazymap. 
Just the original initial nmap scans were kept, I fumbled through making this quickly, so use at your own risk!

Known bugs:
 -I broke the file input feature and haven't gotten around to fixing it. Currently it only scans a subnet or individual IP addresses. For my use case, I just pop open a bunch of terminator windows and run multiple instances.
 -No error handling, if a scan finds nothing it just keeps going.
 -Sloppy nikto handling, just scans from an nmap output pipe of these ports: 80,8000,8080,8081,8888,443,4443

Per the limitations and license, be sure to read what this thing does and use at your own risk.

</p>
<p>Automate NMAP scans and custom Nessus polices.</p>

<h1><a id="user-content-features" class="anchor" aria-hidden="true" href="#features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Features</h1>
<ul>
<li>
<p>Discovers live devices</p>
</li>
<li>
<p>Auto launches port scans on only the discoverd live devices</p>
</li>
<li>
<p>Can run mulitple instances on multiple adaptors at once</p>
</li>
<li>
<p>Creates client Ref directory for each scan</p>
</li>
<li>
<p>Outputs all unique open ports in a Nessus ready format. Much faster to scan in Nessus.</p>
</li>
<li>
<p>Runs as default a T4 speed scan. If you find this too slow, you can press CTRL C in the scan window and it will cleanup and relaunch that one scan with T5 speed option.</p>
</li>
<li>
<p>Logs all start/stop times, live hosts, hosts down, unique ports.</p>
</li>
<li>
<p>Auto creates a custom Nessus policy with only the discovered ports, must faster to scan. *</p>
</li>
<li>
<ul>
<li>Read the script header carefully, in order for the auto Nessus policy creater you must first save a default template to the same directory as the script. The script will detect the default template and create you a unique Nessus policy after each scan for just the unique ports. Then simply import the policy into Nessus and scan just the live devices that the  script found. This will save a massive amount of time scanning, plus will give you more accurate findings.</li>
</ul>
</li>
<li>
<ul>
<li>By Default it will scan a full TCP, Quick UDP, Common ports and a Safe Script scan. You can turn these on and off in the header.</li>
</ul>
</li>
</ul>
